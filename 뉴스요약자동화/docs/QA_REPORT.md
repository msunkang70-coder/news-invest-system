## 5. 개선안 리스트

### 개선 1: fallback 방향성 판단 로직 수정

**WHY:** BULL 97.6% 편향은 시스템 신뢰를 파괴. 가장 긴급.

**HOW:**
- `_fallback_analyze()`에서 제목 기반 감성 판단 강화
- BEAR 키워드에 "급락", "하락", "폭락", "전쟁", "위기", "제재", "감산" 등 한국어 키워드 20개 추가
- 제목에 BEAR 키워드가 1개라도 있으면 BEAR 기본값 (현재는 동점 시 BULL)
- 지정학 뉴스(geo_level 있음)는 기본 BEAR로 설정

**IMPACT:** BULL:BEAR 비율 70:30 수준으로 정상화 예상. 즉시 적용 가능.

### 개선 2: impact_threshold 상향 + 알림 빈도 제어

**WHY:** 일 56건은 알림이 아니라 스팸. 5-10건이 적정.

**HOW:**
- `config.py` IMPACT_THRESHOLD: 5.0 → **6.5** (5-6점대 61건 제거)
- 뉴스 알림 일일 상한: 20 → **10**
- 지표 알림도 동일 일일 상한 풀에 통합
- 고영향(8점+)만 즉시 알림, 6-8점은 배치(2시간)

**IMPACT:** 일 알림 56건 → 10-15건. 즉시 적용 가능.

### 개선 3: LLM 분석 우선순위 적용

**WHY:** 일 20건 할당량을 효율적으로 사용.

**HOW:**
- `summarize_news()`에서 impact_score 내림차순 정렬 후 **TOP 15건만 LLM 호출**
- 나머지는 fallback (개선 1 적용된 버전)
- LLM 실패 시 다음 건으로 넘어감 (현재는 4초 대기 후 fallback)

**IMPACT:** 고영향 뉴스의 시그널/행동 제안 품질 극적 향상. 코드 5줄 수정.

### 개선 4: 뉴스-지표 연결 알림

**WHY:** "환율 급변 + 삼성전자 실적" 같은 복합 이벤트를 하나로 연결.

**HOW:**
- `_dispatch_alerts()`에서 같은 사이클의 뉴스 알림과 지표 알림을 그룹화
- 지표 알림 발송 시 관련 뉴스 TOP 1을 함께 표시
- 예: "🔴 원달러 1,482원 | 관련: 삼성전자 수출 실적 호조 (수출주 수혜)"

**IMPACT:** 교차 분석 자동화. 중간 난이도.

### 개선 5: 초보자 모드 (알림 레벨 분기)

**WHY:** 동일 알림이 전문가와 초보자에게 동시 전달되면 초보자가 이탈.

**HOW:**
- `.env`에 `ALERT_LEVEL=standard|simple` 설정 추가
- simple 모드: "오늘 시장: 강세. 행동: 관망. 주의: 환율 높음" 수준의 1줄 알림
- 전문용어 제거: 에스컬레이션→위험도, 임계값→기준, 승수→배율

**IMPACT:** 사용자층 확대. 중간 난이도.

### 개선 6: 서프라이즈 팩터 실데이터 연동

**WHY:** CPI 3.2% 발표 자체보다 "예상 3.0% 대비 +0.2% 서프라이즈"가 시장 영향이 큼.

**HOW:**
- Investing.com 경제 캘린더 스크래핑으로 컨센서스 수집
- 또는 수동으로 `data/consensus.json`에 주요 지표 예상치 입력 (월 1회)
- `surprise_factor.py`에 연동

**IMPACT:** 매크로 분석가(P3) 가치 극대화. 높은 난이도.
