## 1. 프로젝트 개요

### 1.1 프로젝트 비전

국내외 금융 뉴스, 시장지표(VIX·금리·환율·유가·야간선물), 전쟁·지정학 이벤트, SNS, 경제지표 발표, 전자공시 등을 실시간으로 수집·분석하여 주식 투자에 영향을 주는 핵심 정보를 자동 요약하고, 사용자 맞춤형 이메일 알람을 제공하는 엔드투엔드(End-to-End) 투자 의사결정 지원 시스템을 구축한다.

v1.0 대비 **3대 확장축:**
- **시장지표 실시간 모니터링** — VIX, DXY, 원유, 국채수익률, 야간선물 등 수치 데이터를 직접 수집하여 임계값 기반 알림
- **전쟁·지정학 구조화 분석** — 단순 키워드를 넘어 분쟁 유형·지역·에스컬레이션 단계를 분류하여 시장 영향도 차등 평가
- **교차 자산 영향 체인** — 유가→인플레→금리→주가 등 다중 자산 연쇄 영향을 추적하여 2차·3차 파급효과까지 알림

## 2. 문제 정의 및 연구 동기

### 2.1 현재 시스템의 한계 (v1.0 코드 기반 검증)

**한계 1: 시장지표를 "뉴스 텍스트"로만 감지 — 수치 변동 포착 불가**

현재 코드 (`config.py` KEYWORDS_STRONG)에서 "VIX"는 키워드로 존재하지 않고, "환율", "달러", "유가"는 뉴스 제목/본문에서 텍스트 매칭만 수행한다. 실제 원달러 환율이 1,380원→1,420원으로 급등해도, "환율" 키워드가 포함된 뉴스가 RSS에 올라올 때까지 포착할 수 없다.

```
현재 흐름:  원달러 1,420원 돌파 → (30분~1시간) → 뉴스 기사 작성 → RSS 반영 → 수집
필요 흐름:  원달러 1,420원 돌파 → (즉시) → yfinance/API 감지 → 임계값 알림
```

**한계 2: 코스피 야간선물 완전 미커버**

장 마감(15:30) 후 글로벌 이벤트(미국 CPI 발표, Fed 성명 등)가 발생하면, 야간선물(18:00-06:00)이 실시간 반응한다. 야간선물 -2% 하락은 익일 코스피 하락 가능성이 높다는 강력한 선행 지표이지만, 현재 시스템은 이를 전혀 모니터링하지 않는다.

**한계 3: 지정학 이벤트의 무차별 처리**

현재 코드 (`impact_scorer.py` URGENCY_HIGH)에서 "전쟁", "war", "제재", "sanctions"는 모두 동일한 URGENCY_HIGH로 처리된다. 그러나:
- "북한 미사일 발사 훈련" (레벨 2: 긴장 고조) vs
- "이란-이스라엘 직접 교전" (레벨 4: 무력 충돌) vs
- "대만해협 봉쇄 선언" (레벨 5: 전면 위기)

이들의 시장 영향도는 2배~10배 차이가 나지만, 현재 시스템은 모두 동일하게 취급한다.

**한계 4: 교차 자산 영향 체인 부재**

현재 `signal_aggregator.py`는 개별 뉴스의 시그널을 종목별로 단순 합산한다. "중동 분쟁 → 유가 상승 → 운송비 증가 → 항공/해운주 하락 + 정유주 상승"과 같은 연쇄 영향을 추적하지 못한다.

**한계 5: 경제지표 "서프라이즈 팩터" 미반영**

FRED API로 CPI 값을 수집해도, 시장 예상치(컨센서스) 대비 얼마나 벗어났는지(서프라이즈)를 계산하지 않는다. CPI 3.2% (예상 3.2%)와 CPI 3.5% (예상 3.2%)는 시장 영향이 전혀 다르다.

### 2.2 연구 질문 (v2.0 추가)

### 4.2 아키텍처 설계 원칙 (v2.0 추가)

v1.0의 3대 원칙 (모듈 독립성, 점진적 확장, 장애 격리)에 추가:

**원칙 4: 뉴스-지표 이원화 (Dual-Track Processing)**
뉴스(텍스트)와 시장지표(수치)는 수집-분석 경로가 분리되고, Alert Engine에서 합류한다. 텍스트 분석 장애가 지표 모니터링을 중단시키지 않으며, 그 역도 동일하다.

**원칙 5: 임계값 외부화 (Threshold Externalization)**
모든 시장지표 임계값은 `config.py`에서 중앙 관리하며, 코드 수정 없이 런타임 조정 가능하다.

```python
# models/market_indicator.py (v2.0 신규)
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Optional

class IndicatorCategory(str, Enum):
    VOLATILITY = "변동성"
    FX = "환율"
    COMMODITY = "원자재"
    BOND = "채권"
    EQUITY_INDEX = "주가지수"
    NIGHT_FUTURES = "야간선물"
    SENTIMENT = "심리지표"

class ThresholdLevel(str, Enum):
    NORMAL = "정상"
    WATCH = "주의"       # 관심 구간 진입
    WARNING = "경고"     # 임계값 1단계 돌파
    CRITICAL = "위험"    # 임계값 2단계 돌파
    EXTREME = "극단"     # 역사적 수준

@dataclass
class MarketIndicator:
    """시장지표 데이터 모델"""
    ticker: str                          # ^VIX, CL=F, KRW/USD 등
    name: str                            # "VIX 공포지수"
    category: IndicatorCategory
    current_value: float
    previous_close: float
    change_pct: float                    # 변동률 (%)
    timestamp: datetime

    # 임계값 판단 결과
    threshold_level: ThresholdLevel = ThresholdLevel.NORMAL
    threshold_breached: list[str] = field(default_factory=list)  # 돌파한 임계값 목록
    
    # 시장 영향 방향
    market_implication: str = ""         # "위험자산 회피 심화" 등
    affected_assets: list[str] = field(default_factory=list)  # 영향 받는 자산군

    @property
    def is_alert_worthy(self) -> bool:
        return self.threshold_level in (
            ThresholdLevel.WARNING, ThresholdLevel.CRITICAL, ThresholdLevel.EXTREME
        )
```

def collect_global_indicators() -> list[MarketIndicator]:
    """yfinance를 통한 글로벌 시장지표 수집"""
    indicators = []
    
    tickers = yf.Tickers(" ".join(YFINANCE_TICKERS.keys()))
    
    for ticker_symbol, meta in YFINANCE_TICKERS.items():
        try:
            ticker = tickers.tickers[ticker_symbol]
            info = ticker.fast_info
            hist = ticker.history(period="2d")
            
            if len(hist) < 1:
                continue
            
            current = hist['Close'].iloc[-1]
            prev = hist['Close'].iloc[-2] if len(hist) >= 2 else current
            change_pct = ((current - prev) / prev * 100) if prev != 0 else 0
            
            indicator = MarketIndicator(
                ticker=ticker_symbol,
                name=meta["name"],
                category=meta["category"],
                current_value=round(current, 2),
                previous_close=round(prev, 2),
                change_pct=round(change_pct, 2),
                timestamp=datetime.now(),
            )
            
            # 절대값 임계값 검사
            _check_absolute_thresholds(indicator)
            # 변동률 임계값 검사
            _check_change_thresholds(indicator)
            
            indicators.append(indicator)
            
        except Exception as e:
            logger.warning(f"지표 수집 실패 {ticker_symbol}: {e}")
    
    return indicators

def _check_absolute_thresholds(indicator: MarketIndicator):
    """절대값 기반 임계값 검사"""
    thresholds = THRESHOLDS.get(indicator.ticker, [])
    for threshold_value, level, message in thresholds:
        if indicator.current_value >= threshold_value:
            indicator.threshold_level = level
            indicator.threshold_breached.append(
                f"{indicator.name} {indicator.current_value} ≥ {threshold_value}: {message}"
            )

def _check_change_thresholds(indicator: MarketIndicator):
    """변동률 기반 임계값 검사"""
    change_config = CHANGE_THRESHOLDS.get(indicator.ticker)
    if not change_config:
        return
    
    threshold_pct = change_config.get("pct", 999)
    if abs(indicator.change_pct) >= threshold_pct:
        if indicator.threshold_level.value < change_config["level"].value:
            indicator.threshold_level = change_config["level"]
        indicator.threshold_breached.append(change_config["msg"])

def classify_geopolitical(item: NewsItem) -> GeopoliticalAssessment | None:
    """뉴스의 지정학 이벤트 에스컬레이션 레벨 분류"""
    text = f"{item.title} {item.snippet or ''} {(item.full_text or '')[:500]}".lower()
    
    # 1) 에스컬레이션 레벨 판정 (높은 레벨부터 검사)
    detected_level = None
    for level in sorted(EscalationLevel, reverse=True):
        keywords = ESCALATION_KEYWORDS.get(level, [])
        hits = sum(1 for kw in keywords if kw.lower() in text)
        if hits >= 1:  # L5/L4는 1개만 매칭되어도 충분
            detected_level = level
            break
    
    if detected_level is None:
        return None  # 지정학 이벤트 아님
    
    # 2) 지역 판정
    detected_region = "기타"
    for region, keywords in cfg.KEYWORDS_GEOPOLITICAL.items():
        if region in ("전면전", "무력충돌", "군사행동", "긴장고조", "제재",
                       "에너지무기화", "해상봉쇄"):
            continue  # 분쟁 유형은 스킵
        for kw_list in (keywords if isinstance(keywords, list) 
                        else cfg.KEYWORDS_GEOPOLITICAL.get(region, [])):
            if isinstance(kw_list, str) and kw_list.lower() in text:
                detected_region = region
                break
    
    # 3) 분쟁 유형 판정
    conflict_type = "긴장"
    for ctype in ("전면전", "무력충돌", "군사행동", "긴장고조", "제재"):
        kws = cfg.KEYWORDS_GEOPOLITICAL.get(ctype, [])
        if any(kw.lower() in text for kw in kws):
            conflict_type = ctype
            break
    
    return GeopoliticalAssessment(
        level=detected_level,
        conflict_type=conflict_type,
        region=detected_region,
        impact_multiplier=LEVEL_MULTIPLIERS[detected_level],
        market_channels=REGION_MARKET_CHANNELS.get(detected_region, []),
        description=f"{detected_region} {conflict_type} (Level {detected_level.value})",
    )
```

# analyzers/surprise_factor.py (v2.0 신규)
"""경제지표 서프라이즈 팩터 계산

실제 발표값과 시장 컨센서스의 괴리를 정량화.
서프라이즈가 클수록 시장 영향이 크다.

# 주요 경제지표 컨센서스 (수동 업데이트 또는 API)
# 실전에서는 Investing.com 경제 캘린더 스크래핑 또는 Trading Economics API 활용
CONSENSUS_STORE = {}  # {"CPI_2026_04": {"expected": 3.2, "actual": None}}

def calculate_surprise(indicator_name: str, actual: float, expected: float) -> dict:
    """서프라이즈 팩터 계산"""
    surprise = actual - expected
    surprise_pct = (surprise / expected * 100) if expected != 0 else 0
    
    # 서프라이즈 등급
    abs_surprise = abs(surprise_pct)
    if abs_surprise >= 10:
        grade = "MEGA_SURPRISE"
        impact_boost = 2.0
    elif abs_surprise >= 5:
        grade = "BIG_SURPRISE"
        impact_boost = 1.5
    elif abs_surprise >= 2:
        grade = "MODERATE_SURPRISE"
        impact_boost = 1.2
    else:
        grade = "IN_LINE"
        impact_boost = 1.0
    
    direction = "HAWKISH" if surprise > 0 else "DOVISH"  # CPI 기준
    
    return {
        "actual": actual,
        "expected": expected,
        "surprise": round(surprise, 2),
        "surprise_pct": round(surprise_pct, 2),
        "grade": grade,
        "impact_boost": impact_boost,
        "direction": direction,
        "description": f"{indicator_name}: {actual} (예상 {expected}, "
                       f"서프라이즈 {surprise:+.2f}, {grade})"
    }
```

### 7.3 기능 3: 야간선물 → 익일 전망 자동 생성 (v2.0 신규)

```python
# analyzers/overnight_outlook.py (v2.0 신규)
def generate_overnight_outlook(
    night_futures: MarketIndicator,
    overnight_news: list[NewsItem],
    global_indicators: list[MarketIndicator],
) -> dict:
    """야간선물 + 글로벌 지표 + 야간 뉴스 → 익일 코스피 전망"""
    
    # 1) 야간선물 신호
    futures_signal = "강세" if night_futures.change_pct > 0.5 else \
                     "약세" if night_futures.change_pct < -0.5 else "보합"
    
    # 2) 글로벌 지표 신호
    sp500 = next((i for i in global_indicators if i.ticker == "^GSPC"), None)
    vix = next((i for i in global_indicators if i.ticker == "^VIX"), None)
    
    # 3) 야간 뉴스 종합
    bull_count = sum(1 for n in overnight_news if n.direction == Direction.BULL)
    bear_count = sum(1 for n in overnight_news if n.direction == Direction.BEAR)
    
    return {
        "outlook": futures_signal,
        "night_futures_change": f"{night_futures.change_pct:+.2f}%",
        "sp500_change": f"{sp500.change_pct:+.2f}%" if sp500 else "N/A",
        "vix_level": f"{vix.current_value:.1f}" if vix else "N/A",
        "news_sentiment": f"BULL {bull_count} vs BEAR {bear_count}",
        "summary": f"야간선물 {night_futures.change_pct:+.1f}%, "
                   f"S&P500 {sp500.change_pct:+.1f}% → 익일 코스피 {futures_signal} 전망"
    }
```

## 14. 윤리적 고려사항 및 안전성

v1.0과 동일하게 유지하되, 지정학 분류에 대한 추가 고려사항:

**원칙 5: 지정학 분석의 중립성 (Neutrality)**
지정학 이벤트 분류 시 정치적 입장을 반영하지 않는다. "어느 쪽이 옳은가"가 아닌 "시장에 어떤 영향을 미치는가"에만 집중한다. 에스컬레이션 레벨은 순수하게 군사적/경제적 영향도 기준으로 판정한다.
